# 跨平台免服务打包说明

本文档只说明项目当前保留的最终打包方式：跨平台免服务版。

## 目标

打包后的产物可以直接双击 `index.html` 打开，不依赖本地 HTTP 服务，不区分 Windows 和 macOS。

当前方案基于以下配置：

- `vite` 负责生产构建
- `vite-plugin-singlefile` 将 JS 和 CSS 内联到 `dist/index.html`
- `base: './'` 保证静态资源使用相对路径

这样生成的页面可以直接在 `file://` 下运行，避免普通 Vite 产物在本地双击时出现空白页。

## 前提

在项目根目录 `prompt-randomizer` 下执行命令。

已安装依赖时，目录中应存在 `node_modules`。如果尚未安装，先执行：

```powershell
npm install
```

## 构建应用

执行：

```powershell
npm run build
```

构建完成后，离线可运行产物位于：

- `dist/index.html`
- `dist/favicon.svg`
- `dist/icons.svg`

其中 `dist/index.html` 是最终入口文件，可直接双击打开。

## 生成发布包

在项目根目录执行以下 PowerShell 命令：

```powershell
if (Test-Path ".\便携包-免服务") { Remove-Item ".\便携包-免服务" -Recurse -Force }
New-Item -ItemType Directory -Path ".\便携包-免服务" | Out-Null
Copy-Item -Path ".\dist\*" -Destination ".\便携包-免服务" -Recurse -Force
if (Test-Path ".\提示词随机组合器-跨平台免服务版.zip") { Remove-Item ".\提示词随机组合器-跨平台免服务版.zip" -Force }
Compress-Archive -Path ".\便携包-免服务\*" -DestinationPath ".\提示词随机组合器-跨平台免服务版.zip" -CompressionLevel Optimal
```

生成后会得到两个发布产物：

- `便携包-免服务/`
- `提示词随机组合器-跨平台免服务版.zip`

## 验证方式

打包后直接双击以下文件进行验证：

```text
便携包-免服务/index.html
```

如果页面能正常显示“提示词随机组合器”界面，就说明免服务包可用。

## 注意事项

- 本项目只保留这一种打包方式，不再区分 Windows 版和 macOS 版启动脚本。
- 不要再使用“启动本地服务器再打开浏览器”的旧方案作为默认发布方式。
- 如果修改了前端代码，必须重新执行 `npm run build`，再重新生成 `便携包-免服务` 和 zip 文件。
